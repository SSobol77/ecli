# ECLI v0.2.3 Release Notes

ECLI v0.2.3 is a Panel Console Stabilization release. It keeps the
Services Foundation behavior intact while replacing the fragile F11 terminal
experiment with an ECLI-owned PySH Console Panel.

## Highlights

- Full PTY terminal emulator scope is rejected for ECLI 0.2.x.

- F11 opens or focuses the ECLI-owned PySH Console Panel.

- F12 focus switching between editor and panels is documented in F1 Help.

- The PySH backend uses subprocess argv form, not shell interpolation.

- Missing PySH backend diagnostics are deterministic.

- Console builtins are available for `cd`, `pwd`, `clear`, and `exit`.

- Command cancellation is available through Ctrl+C.

- Regression coverage exists for the PySH backend, console panel, keybindings,
  F1 Help discoverability, and forbidden PTY implementation paths.

## Non-Goals and Safety Boundaries

- No full terminal emulator.

- No VT parser.

- No raw interactive PySH embedded inside curses.

- No fullscreen terminal application support target.

- No PySH source migration.

- No monorepo conversion.

- No VMLab, QEMU, or QMP runtime scope.

- No privileged remediation.

## Validation

Release validation for this update uses:

```sh
uv run ruff check src tests
uv run pytest -q
uv run python scripts/check_runtime_imports.py
find src/ecli -maxdepth 4 -type f | sort | grep -E 'command_backends|terminal_screen.py|terminal_session.py' || true
```

The forbidden PTY path scan must produce no output.
