# ECLI v0.2.1 Release Notes

## Release Theme

ECLI v0.2.1 is a release-hardening update. It keeps the existing editor and
Services Foundation behavior intact while fixing source-text correctness,
Python package assets, desktop launcher registration, and Linux package
coverage for the release pipeline.

## Editor Correctness

- Opening existing source files now preserves leading whitespace, tabs, mixed
  indentation, blank lines, blank lines containing spaces, and trailing spaces.
- Saving an unmodified UTF-8 LF text file does not rewrite file bytes.
- Syntax highlighting keeps source indentation visible instead of shifting
  indented code to column 1.
- Valid UTF-8 source files are decoded as UTF-8 before any detector fallback.
  This prevents Cyrillic comments and docstrings from being misdecoded as
  MacRoman.

## Python Package Assets

- The canonical packaged icon is `src/ecli/assets/ecli.png`.
- Wheel and source distributions include the icon.
- Runtime icon lookup uses package resources rather than repository-root paths.
- `ecli-install-desktop-entry` installs a user-level Linux desktop launcher and
  icon for `pip` and `pipx` installs without requiring `sudo`.

## Linux Packaging

- SUSE/openSUSE RPM support is available through the release packaging scripts.
- Slackware package support emits `.txz` packages.
- Arch Linux support includes `packaging/arch/PKGBUILD`.
- The Arch release script normalizes raw `makepkg` output to the ECLI release
  artifact naming contract.
- Nix/NixOS support includes a flake and local package expression for `nix run
  .`, `nix build .`, and `nix profile install .`.

## Artifact Naming Contract

GitHub Release artifacts use normalized ECLI names:

```text
ecli_<version>_<platform>_<arch>.<extension>
```

The Linux package family includes:

```text
ecli_<version>_linux_x86_64.deb
ecli_<version>_linux_x86_64.rpm
ecli_<version>_opensuse_x86_64.rpm
ecli_<version>_arch_x86_64.pkg.tar.zst
ecli_<version>_slackware_x86_64.txz
ecli_<version>_linux_x86_64.AppImage
```

Raw Arch `makepkg` output may use
`ecli-editor-<version>-1-<arch>.pkg.tar.zst`; ECLI release artifacts normalize
that package to `ecli_<version>_arch_<arch>.pkg.tar.zst`.

## Safety Boundaries

- SystemDoctor remains read-only.
- CommandPlan output remains draft/preview-only.
- PrivilegedActionService does not perform real privileged execution.
- Service panels do not execute remediation.
- No VMLab runtime behavior is included.
