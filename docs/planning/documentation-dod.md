# Documentation Definition of Done

- [ ] Documentation hierarchy is layered and category-based.
- [ ] Current-state and target-state architecture are separated.
- [ ] Artifact contract is explicit and authoritative.
- [ ] Contributor docs contain no placeholder TODO sections.
- [ ] Config behavior and precedence are documented.
- [ ] Testing and CI quality expectations are documented.
- [ ] Extension boundaries are documented.
- [ ] Migration map old -> new exists and is maintained.
- [ ] Archive policy exists and superseded docs are non-authoritative.
